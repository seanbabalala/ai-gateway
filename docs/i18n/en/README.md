# SiftGate Documentation

[Documentation home](../../README.md) · [Project README](../../../README.md)

SiftGate is a self-hosted AI traffic gateway for teams running agents and
applications across multiple providers. It keeps routing, policy, budgets,
audit, and metadata evidence local by default.

## Quick Start

```bash
git clone https://github.com/seanbabalala/ai-gateway.git
cd ai-gateway
npm install
cd frontend && npm install && cd ..
cp gateway.config.example.yaml gateway.config.yaml
cp .env.example .env
npm run build
npm start
```

Open the Dashboard at `http://localhost:2099/dashboard`, add one Provider Node,
create a Gateway API Key, and send your first request to
`http://localhost:2099/v1/chat/completions`.

## First-Run Path

1. Confirm the active Workspace.
2. Add a Provider Node.
3. Create a Gateway API Key.
4. Optionally bind a Policy Namespace.
5. Review daily Budget scope.
6. Send a first request.
7. Inspect Logs and Route Explanation.
8. Configure advanced surfaces only when needed.

## Key Concepts

| Concept | Meaning |
| --- | --- |
| Workspace | Local Dashboard and metadata boundary. |
| Provider Node | Configured upstream account, deployment, proxy, or local runtime. |
| Gateway API Key | Client-facing key generated by SiftGate. |
| Policy Namespace | Config-backed local policy label for keys, teams, budgets, node/model allow-lists, and limits. |
| MCP Tool Gateway | Tool-call governance and proxying, not model routing. |
